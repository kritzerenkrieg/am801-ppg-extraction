AM806 Finger Pulse Oximeter
Communication Protocol            
1.1.	Overview
This communication protocol applies to the high-sampling-rate finger pulse oximeters developed and customized by MedLinket Corporation.
1.2.	Communication Interface Description
The device or module can communicate with external devices such as computers (or tablets, mobile phones) via serial port (or Bluetooth). The AM801 developer version additionally supports direct USB connection communication. For details, please refer to the specification of the device or module.
1.3.	Serial Port Parameter Settings
AM-801 Model：Baud Rate 9600，8 Data Bits，None Parity，1 Stop Bit。

2.	General Data Packet Format
head 1	head 2	length	check sum	device ID	CMD ID	DATA
0xFF	0xFE	LL	CS	0x23	CMD	DATA

Data Packet Description：
1)The header of the data packet consists of 2 bytes: 0xFF and 0xFE respectively.
2）Total length of the packet excluding the header, i.e., total packet bytes - 2.
3）The sum of all bytes except the header and length, reserved as a single byte, truncated if exceeding 1 byte.
Example：CS = (LL+0x23+0x95+sum(Data)) & 0xFF
4）Indicates the device type; 0x23 stands for pulse oximeter/oximeter module.
5)Indicates the command type of the data packet, with the corresponding relationship as follows:
CMD	Command Type
0x95	Measurement Result Packet
0x96	PPG Waveform Packet
0x97	HRV Data Packet
0x99	Battery Level Packet
0x85	Raw Data Packet
0x1D	UID Transmission Command
0xB0	Device Control Command

DATA：Consists of 0 or more bytes for transmitting data information, whose meaning is related to CMD.
Example：When CMD=0x95, DATA contains the information of blood oxygen and pulse rate.

2.3. Heart Rate Variability (HRV) Data 0x97（HRV data）
(head 1)	(head 2)	(length)	(check sum)	(device ID)	(CMD ID)	DATA
0xFF	0xFE	0x18	CS	0x23	0x97	（20 Bytes）

Packet 0x97: Contains HRV data, representing the time interval between heartbeats, measured in milliseconds (ms).
Transmission frequency: HRV data is transmitted in packets every 10 heartbeat waveforms; each HRV data packet consists of 2 bytes.

2.4.	UID Query and Transmission Command 0x1D
Host Query Command: The host queries the device's Unique ID (UID) by sending this command, and the device transmits the UID information upon receiving the command.
（Query the unique ID）
(head 1)	(head 2)	(length)	(check sum)	(device ID)	(CMD ID)	/
0xFF	0xFE	0x04	CS	0x23	0x1D	/



Device UID Information Transmission Packet：
（Send the unique ID）

(head 1)	(head 2)	(length)	(check sum)	(device ID)	(CMD ID)	UID DATA
0xFF	0xFE	0x14	CS	0x23	0x1D	（16Bytes）

2.5.	Device Data Transmission Control Command 0xB0

（Control the device to send data）
(head 1)	(head 2)	(length)	(check sum)	(device ID)	(CMD ID)	DATA
0xFF	0xFE	0x05	CS	0x23	0xB0	（1Bytes）
0：（Stop），
1：（Start）

Command 0xB0: Host controls the device to start/stop data transmission to the host via this command, device executes it after reception.
Default setting: Data transmission enabled.
